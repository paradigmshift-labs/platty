# MCP SDD 사용자 스토리 형식

`user_stories.md`는 한국어 사용자 시나리오 문서다. 구현 근거와 내부 검토 기록은
`prd.md` §9에 둔다.

## Frontmatter

```yaml
---
id: "SPEC-<slug>-<YYYY-MM>"
type: "sdd-stories"
status: "draft"
projectId: "<projectId>"
outputLanguage: "ko"
derivedFrom: "prd.md"
---
```

소스·최신성·impact·Self Review 메타데이터는 반복하지 않는다. 사용자 스토리는
제품 입력 연결에 필요한 최소 header만 유지하고 상세 근거는 `prd.md` §9를 따른다.

## 본문

```markdown
# 사용자 스토리 — <요청 제목>

> 사용자 관점의 결과와 검증 가능한 시나리오만 적는다.

## 스토리 한눈에 보기

| 스토리 | 사용자 | 기대 결과 | 연결 규칙·수용 기준 | 시나리오 | 영향을 주는 가정·미결 질문 |
| --- | --- | --- | --- | --- | --- |
| US-01 | | | R-01 / AC-01 | US-01-S01, US-01-S02 (2개) | A-01 / O-01 |

## US-01. <스토리 제목>

**사용자로서** <사용자>는<br>
**<목표>를 위해** <행동>하고 싶다.<br>
**그래서** <기대 결과>를 얻는다.

### US-01-S01. 정상 흐름

- **Given** <상태>
- **When** <행동 또는 트리거>
- **Then** <사용자/운영자가 확인하는 결과>

### US-01-S02. 예외 흐름

- **Given** <예외 상태>
- **When** <행동 또는 트리거>
- **Then** <기대 결과>

## 규칙·수용 기준·시나리오 연결

| 제품 규칙 | 수용 기준 | 사용자 스토리 | 시나리오 | 확정 결정 | 남은 가정·미결 질문 |
| --- | --- | --- | --- | --- | --- |
| R-01 | AC-01 | US-01 | US-01-S01 | D-01 | A-01 / O-01 |
| R-01 | AC-01 | US-01 | US-01-S02 | D-01 | A-01 / O-01 |

## 가정·미결 질문 영향

| ID | 상태·소유자 | 영향을 받는 스토리·시나리오 | 현재 적용한 가정 | 답이 바뀌면 재검토할 것 |
| --- | --- | --- | --- | --- |
| O-01 | open / <소유자> | US-01 / US-01-S02 | A-01 | R-01 / AC-01 |
```

## 작성 규칙

- `스토리 한눈에 보기`는 상세 스토리와 연결표에서 파생한 탐색 목차다. 새 스토리,
  새 제품 결정, 새 근거를 만들지 않으며 스토리·시나리오가 바뀌면 함께 갱신한다.
- 한눈에 보기에는 범위 축약 없이 실제 시나리오 ID를 쉼표로 모두 쓰고, 개수도 상세
  시나리오와 정확히 일치시킨다.
- 각 스토리와 시나리오는 `R-*` 규칙과 `AC-*` 수용 기준에 연결한다.
- 모든 시나리오는 연결 표에 하나 이상의 행을 가지며, 하나의 행에 여러 시나리오를
  축약해서 쓰지 않는다.
- Given/When/Then은 사용자가 관찰하는 조건·행동·결과만 쓴다. route, component,
  class, handler, file, line, code node, spec id, source parity와 검색 후보는 시나리오나
  가정 표에 넣지 않고 `prd.md` §9 또는 후속 `system_design.md`로 보낸다.
- 기존 화면이나 흐름을 재사용하는 스토리는 `기존 호스트 승인 화면`, `기존 결제 흐름`처럼
  사용자에게 보이는 목적만 쓴다. 정확한 구현 이름을 확인하는 일은 제품 가정이 아니라
  설계의 `TQ-*` 또는 Evidence-Resolution이다.
- `US-NN-SNN` 식별자는 한 번 부여하면 순서 변경으로 다시 번호를 매기지 않는다.
- 시나리오 제목 자체에 `US-NN-SNN`을 쓰고 연결 표의 ID와 정확히 일치시킨다.
- `A-*` 또는 `O-*`가 시나리오를 바꿀 수 있으면 영향 표에 소유자·상태·영향 ID와
  현재 적용한 가정을 기록한다.
- 내부 구현 방식, MCP 도구 호출, 근거 매트릭스, Self Review는 본문에 넣지 않는다.
